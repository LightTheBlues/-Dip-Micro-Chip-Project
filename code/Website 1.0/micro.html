Background

A Controller Area Network bus (CAN bus for short) is an asynchrnounous communication protocol that enables communication between CAN capable devices without the need for a host computer. Several versions of CAN were developed in the late 80s, but it was CAN 2.0A and CAN 2.0B that soon became the industry standard for automotive applications. Today, the most commonly used version for CAN implementations is CAN-FD (Flexible Data rate), which is backwards compatible with 2.0.

Our Goal

  • Microchip, which is known for the innovative company and has the provision of embedded - and automobile - control systems requests us to implement the specific firmware on dsPIC33CSC. This inspires us to acknowledge the importance of efficient and reliable communications.

  • Microchip Technology provides an Integrated Development Environment (IDE) known as MPLAB-X, offering powerful features such as visualized I/O operations, code debugging, and validation of embedded designs for microcontrollers and digital signal controllers.

  • This tool allows for the implementation and testing of CAN communication codes compatible with the dsPIC33C family
  • Controller Area Network (CAN) is a data communication protocol utilized for broadcasting data and managing control systems.

  • The CAN protocol is a cost-effective communication option with real-time capabilities suitable for complex electrical environments.Various boards support the CAN protocol.

  • CAN is versatile and finds applications in a wide range of fields. While initially designed for interconnecting automotive devices, it has expanded into various industries. Crucial use-cases include agricultural equipment, navigational equipment, mechanical control, aircraft fueling systems, medical instruments, maritime applications such as ships, lighting control systems, and robotics. For example, medical equipment manufacturers have integrated CAN into medical devices, and hospitals utilize CAN-based systems to control room components like lights, cameras, and X-ray machines.
